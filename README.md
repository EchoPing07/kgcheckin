# 酷狗概念版 VIP 自动领取

基于青龙面板的酷狗概念版 VIP 每日自动签到工具。

## 功能特性

- **每日自动签到**：听歌领取 + 看广告领取
- **多账号支持**：一个环境变量管理多个账号，用 `@` 分隔
- **Token 自动续期（可选）**：配置 OpenAPI 凭证后，每周日自动刷新 token 并回写环境变量；未配置则跳过刷新，避免 token 失效（可选）
- **日志脱敏**：自动隐藏 token、昵称、userid 等敏感信息
- **通知推送**：签到结果通过青龙通知渠道推送
- **零 HTTP 服务**：直接函数调用，不启动 Express 服务器，内存占用极低

## 部署步骤

### 第一步：添加订阅

进入青龙面板，点击左侧菜单「订阅管理」，点击右上角「新建订阅」：

| 字段     | 填写内容                                          |
| ------ | --------------------------------------------- |
| 类型     | 公开仓库                                          |
| 链接     | `https://github.com/EchoPing07/kgcheckin.git` |
| 分支     | `main`                                        |
| 文件后缀   | 留空（默认拉取全部文件）                                  |
| 定时类型   | 按需选择                                          |
| 自动添加任务 | 关闭                                            |

填写完成后点击「确定」，然后点击「运行」按钮手动拉取一次。等待状态变为「完成」。

> **手动部署：** 如果不方便使用订阅，也可以将仓库文件手动放入青龙的目录下。

### 第二步：安装依赖

本脚本只需要一个 Node.js 依赖：**axios**。

进入青龙面板，点击左侧菜单「依赖管理」：

1. 点击右上角「新建依赖」
2. 依赖类型选择 **NodeJs**
3. 依赖名称填写：`axios`
4. 自动拆分：选择「不拆分」
5. 点击「确定」
6. 回到依赖列表，找到刚添加的 axios，点击「安装」按钮

等待安装状态变为「已安装」即可。

> 青龙会将 axios 安装到全局 Node.js 环境中，脚本可以直接 require 引用。根目录不需要额外安装任何依赖。

### 第三步：获取账号凭证

你需要获取酷狗概念版的 `userid` 和 `token`。步骤如下：

**1. 安装酷狗概念版**

在手机安装并登录酷狗概念版。

**2. 使用抓包工具抓取凭证**

根据你的设备选择抓包工具：

- **iOS**：Stream、Shadowrocket（自带抓包）、HttpCatcher
- **Android**：HttpCanary、Packet Capture、小黄鸟（需 root）

打开抓包工具开始抓包，然后打开酷狗概念版随便点几下，在抓包记录中搜索 `gateway.kugou.com` 相关的请求。

**3. 从请求中提取 userid 和 token**

在请求的 Cookie 或 Header 中找到以下字段：

```
token=qwertyuiop1234567890...（一长串字符）
userid=0000114514
```

> 也可以使用其他第三方工具获取，以实际情况按需选择

### 第四步：配置环境变量

进入青龙面板，点击左侧菜单「环境变量」，点击「新建变量」：

| 变量名                | 必填  | 说明                               |
| ------------------ | --- | -------------------------------- |
| `KUGOU_CK`         | 是   | 账号凭证，格式见下方                       |
| `KG_PLATFORM`      | 否   | 固定填 `lite` ，不填默认也是 lite          |
| `QL_CLIENT_ID`     | 否   | 青龙 OpenAPI 客户端 ID（用于 token 自动续期和回写） |
| `QL_CLIENT_SECRET` | 否   | 青龙 OpenAPI 客户端密钥（用于 token 自动续期和回写）  |
| `KUGOU_API_PROXY`  | 否   | HTTP 代理地址                        |

**KUGOU_CK 格式：**

单账号：

```
userid#token
```

示例：

```
0000000001#asdfghjkl0123456789
```

多账号（用 `@` 分隔）：

```
userid1#token1@userid2#token2@userid3#token3
```

> **注意：** token 中不要包含空格。如果你的 token 来自完整 Cookie 字符串，确保只取 `token=` 后面到下一个 `;` 之前的部分。

### 第五步：创建定时任务

进入青龙面板，点击左侧菜单「定时任务」，点击「新建任务」：

| 字段   | 填写内容                                          |
| ---- | --------------------------------------------- |
| 名称   | `酷狗概念签到`                                      |
| 命令   | `task EchoPing07_kgcheckin_main/kgcheckin.js` |
| 定时规则 | `10 1 * * *`                                  |

关于定时规则的说明：

- `10 1 * * *` 表示每天北京时间 **01:10** 执行
- 格式为 `分 时 日 月 周`，青龙面板使用北京时间（UTC+8）
- 不建议设置太早（如 00:00），服务器可能在重置中

> **任务命令格式说明：** `task` 是青龙内置的命令，会自动在 `/ql/data/repo/` 和 `/ql/data/scripts/` 下查找脚本。如果你的青龙版本不支持 `task` 命令，可以改用完整路径：`node /ql/data/repo/EchoPing07_kgcheckin_main/kgcheckin.js`

创建完成后点击「确定」，然后点击「运行」手动执行一次，查看日志确认签到是否正常。

### 第六步：Token 自动续期（推荐）

为了延长酷狗概念版的 token 使用寿命，脚本支持每周日自动刷新 token。**但刷新操作只有在配置了 OpenAPI 凭证时才会执行**——因为刷新后旧 token 会失效，如果无法将新 token 写回环境变量，下次运行时就会签到失败。

如果你不配置凭证，脚本会跳过刷新，当前 token 在有效期内可以一直正常使用。token 过期后需要重新抓包获取。

**1. 创建 API 应用**

进入青龙面板「系统设置」→「应用设置」→ 点击「新建应用」：

- 名称：随意填写，如 `kgcheckin`
- 权限：至少勾选 **「环境变量」**（读取和修改）
- 点击确定

**2. 配置凭证**

创建后会显示 `Client ID` 和 `Client Secret`，将它们分别填入青龙的环境变量：

- `QL_CLIENT_ID` = 你的 Client ID
- `QL_CLIENT_SECRET` = 你的 Client Secret

**3. 工作流程**

配置凭证后，脚本每周日运行时会：刷新 token → 将新值打印到日志 → 通过 OpenAPI 写回 `KUGOU_CK` → 回读验证是否写入成功。如果验证失败，会通过通知推送新 token 值，方便你手动更新。

## 日志与通知

### 日志格式

脚本使用带时间戳和前缀的日志格式，方便在青龙日志中查看：

```
[2026/7/10 01:10:05] [INFO]    ========== 酷狗概念 VIP 每日签到 ==========
[2026/7/10 01:10:05] [INFO]    共 2 个账号
[2026/7/10 01:10:05] [INFO]    账号1 验证 token...
[2026/7/10 01:10:06] [INFO]    账号1 [张**三] 开始签到
[2026/7/10 01:10:06] [INFO]    账号1 听歌领取 VIP...
[2026/7/10 01:10:06] [SUCCESS] 账号1 听歌领取成功
[2026/7/10 01:10:07] [SUCCESS] 账号1 第1次广告领取成功
[2026/7/10 01:10:37] [INFO]    账号1 广告奖励今日已领取
[2026/7/10 01:10:37] [INFO]    账号1 VIP 到期时间: 2026-07-12 01:10:06
[2026/7/10 01:10:37] [INFO]    账号1 [张**三] 签到完成
```

日志中的昵称和 userid 会自动脱敏（如 `张**三`、`204***16`），token 等敏感字段会显示为 `[REDACTED]`。

### 通知推送

脚本会在以下情况发送通知：

- **全部签到成功**：推送成功摘要
- **存在异常**：推送错误详情，方便排查
- **Token 回写失败**：推送提醒手动更新 token
- **脚本异常退出**：推送异常信息

通知通过青龙内置的 `sendNotify` 机制推送（需在青龙「系统设置 → 通知设置」中配置）。

## 常见问题

### 签到失败：token 过期或账号不存在

token 非长期有效，过期后需要重新抓包获取新 token，更新 `KUGOU_CK` 环境变量。如果配置了 `QL_CLIENT_ID` 和 `QL_CLIENT_SECRET`，脚本每周日会自动刷新 token 并回写，可以大幅延长使用寿命。未配置凭证时脚本不会刷新 token，避免刷新后无法回写导致当前 token 提前失效。

### 听歌/广告领取失败：error_code 130012 / 20028 / 30002

这些都是正常的「已完成」状态码，脚本已正确处理：

- `130012`：听歌今日已领取
- `20028`：广告奖励今日已领取
- `30002`：今日广告次数已用完

如果脚本日志中显示了这些信息，说明当天已经签到过了，无需担心。

### 依赖安装失败：Cannot find module 'axios'

进入青龙「依赖管理」，确认 axios 已安装成功（状态为「已安装」）。如果安装失败，可以尝试重新安装或检查青龙容器的网络连接。

### 任务执行报错：task 命令找不到

部分青龙版本的 `task` 命令路径不同，尝试将任务命令改为：

```
node /ql/data/repo/EchoPing07_kgcheckin_main/kgcheckin.js
```

或者：

```
node /ql/repo/EchoPing07_kgcheckin_main/kgcheckin.js
```

根据你的青龙数据目录实际路径调整。

### 多个账号只有第一个签到成功

检查 `KUGOU_CK` 格式是否正确，确保账号之间用 `@` 分隔，且每个账号的 userid 和 token 之间用 `#` 分隔，不要有多余空格。

正确示例：

```
0000000001#asdf...789@0000000002#qwer...789
```

## 项目结构

```
kgcheckin/
├── kgcheckin.js          # 青龙定时任务入口脚本
├── kugouApi.js           # 酷狗 API 封装层（直接调用模块，无 HTTP 服务）
├── qlApi.js              # 青龙 OpenAPI 交互（token 回写 + 通知推送）
├── utils.js              # 工具函数（日志、脱敏、cookie 解析）
├── package.json          # 项目配置（根目录无额外依赖）
├── README.md
└── api/                  # 酷狗 API 底层模块（CommonJS）
    ├── package.json      # 仅依赖 axios
    ├── module/           # 5 个签到相关接口模块
    │   ├── user_detail.js          获取用户信息
    │   ├── login_token.js          刷新登录 token
    │   ├── youth_listen_song.js    听歌领取 VIP
    │   ├── youth_vip.js            看广告领取 VIP
    │   └── user_vip_detail.js      查询 VIP 到期时间
    └── util/             # 加密、签名、请求工具
        ├── config.js               酷狗客户端配置（appid 等）
        ├── crypto.js               MD5 / AES / RSA 加解密
        ├── helper.js               请求签名生成
        ├── request.js              HTTP 请求封装（axios）
        ├── runtime.js              代理配置解析
        └── util.js                 cookie 工具
```
